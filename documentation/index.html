RVDM Angular Starterkit

Angular Boilerplate with Redux (ngrx), Lazy Loading, SSR (Server Side Rendering), PWA(Progressive Web App), Unit Test (Karma), E2E Test (Protractor / Cucumber)

This project was generated with Angular CLI version 8.x.x. On default this build is compiled with AOT (Ahead of Time compiler)

Quickstart: npm run start:clean

Features

  • Angular Boilerplate with Redux (ngrx)
  • Lazy Loading
  • NGX-Translate (translations from assets/i18n/{locale}.json)
  • Prerendering
  • PWA (Progressive Web App)
  • SSR (Server Side Rendering)
  • Unit Test (Karma)
  • E2E Test / Reports (Protractor / Cucumber)
  • Documentation Generation (Compodoc)
  • Google Lighthouse reporter (save to Compodoc additional docs)

example sites:

  • Progressive Web App (PWA), NGRX and lazy loading: https://angular.rickvandermeij.nl
  • Progressive Web App (PWA), NGRX, lazy loading, and Server Side Rendering: https://angular-ssr.rickvandermeij.nl

Development server

Run npm start for a dev server. Navigate to https://localhost:4202/. The app will automatically reload if you change any of the source files.

Because we are running localhost on SSL (https), you will need to generate a certificate and key, and place them in a folder called build. Run npm run certificate:generate to create the required certificates and place them in the required folder. Another way to create those can be found here. The build map should already be in your .gitignore.

When developing, Husky is implemented, which means you only can commit when there are no linting errors, and all unit-tests succeeds. Other commmands can be implemented in the package.json.

Build

Run npm run build to build the project. The build artifacts will be stored in the dist/ directory. Use the --prod flag for a production build. When building all ununsed CSS will be removed by the purgeCSS plugin.

Run npm run build:gzip to build the project and then gzip the files if neccesary.

Ivy Build

To create an Ivy build, you'll need to update src/tsconfig.app.json and set enableIvy: true. Be aware when doing this, functionality for SSR and Prerendering won't work at this moment. Maybe when updating in the future, it will.

Extend the Angular CLI settings (Webpack)

It is possible to extend the Angular CLI settings via a Webpack partial. Angular CLI will still optimize all functionality without being ejected. The Webpack Partial still gives you the opportunity the specify certain extra configurations in Webpack.

Server Side Rendering (SSR)

This project comes with built-in SSR functionality. The effect of SSR will be valuable on larger projects or slower internet connections, run Google Audit with slow network to see the difference.

  • For Serving, run npm run start:ssr
  • For Building, run npm run ssr:build

Prerendering Angular

  • For Serving, run npm run start:prerender
  • For Building, run npm run prerender:build

Testing

Run npm run lint for linting your application.

Running unit tests with Karma

Run npm run test to execute the unit tests via Karma. This script will be run as a watcher. Most effective to run this alongside npm run start while developing.

Run test:unit-headless to execute a single run for the Unit test. Best usecases are for pre-commit checks and in pipeline scripts.

Both scripts will provide a Code Coverage file, which can be found in './reports/coverage'

Running E2E tests with Protractor and Cucumber

Run npm run test:e2e to execute the E2E protractor tests. Tests can be found within ./features all written in Cucumber style.

This test will provide an report which will be shown at the end of all tests. It can also be found in ./reports/e2e/report. When tests fails, there will be a screenshot attached to the scenario where the test has failed.

Analyzing

Run npm run wpo:stats to get a clear view of all dependencies and their dependencies for your project.

Documentation

Run npm run doc to generate a compodoc containing documentation for all the modules, routes and other components of the application.
